# SP-Capture2-P1 真机探针结论(F2SP / SDJW-F2SP,MT6768,Android 13)

> 探针 `debug/Capture2Probe.kt`,由 Diagnostics 屏临时按钮触发,logcat 标签 `CAP2PROBE`。
> 真机跑于 2026-07-14,设备 F2S202503103054。**结论:P2 全绿放行。**

## 1. 传感器 / 相机

- 后置 `cameraId=0`,`INFO_SUPPORTED_HARDWARE_LEVEL=1`(**FULL**)。
- `SENSOR_INFO_ACTIVE_ARRAY_SIZE=Rect(0,0-2592,1944)`,`PIXEL_ARRAY=2592x1944`(**5MP,原生 4:3**)。

## 2. 各流可用尺寸(StreamConfigurationMap)

三档(VIDEO=MediaRecorder / JPEG / PREVIEW=SurfaceTexture)可用尺寸基本一致:

| 流 | 最大 | 最大 4:3 | 最大 16:9 |
|---|---|---|---|
| VIDEO(相机输出面) | 2592×1944 | **2592×1944** | **2560×1440** |
| JPEG(拍照) | 2592×1944 | **2592×1944** | 2560×1440 |
| PREVIEW | 2592×1944 | 2592×1944 | 2560×1440 |

常用 4:3 档:2592×1944 / 1920×1440 / 1440×1080 / 1280×960 / 960×720 / 640×480。
常用 16:9 档:2560×1440 / 1920×1080 / 1280×720。

**关键**:相机输出面本身能给到满 5MP;**瓶颈在编码器,不在相机**。

## 3. 硬件编码器能力(MediaCodecList)

| MIME | 编码器 | 宽范围 | 高范围 | 码率上限 |
|---|---|---|---|---|
| video/hevc | **c2.mtk.hevc.encoder** | [160, **1920**] | [128, **1088**] | 50 Mbps |
| video/hevc | OMX.MTK.VIDEO.ENCODER.HEVC | [160, 1920] | [128, 1088] | 50 Mbps |
| video/hevc | c2.android.hevc.encoder(软) | [2, 512] | [2, 512] | 10 Mbps |
| video/avc | **c2.mtk.avc.encoder** | [128, **1920**] | [128, **1088**] | 168 Mbps |
| video/avc | OMX.MTK.VIDEO.ENCODER.AVC | [128, 1920] | [128, 1088] | 168 Mbps |

**硬编上限 = 1920×1088**(HEVC 与 AVC 同)。故:

- **视频编码天花板 = 1080p 级**(4:3 → 1440×1080;16:9 → 1920×1080)。想上 2K/满5MP 视频只有软编 c2.android.*(512² 上限,不可用),**放弃满5MP 视频**。
- **拍照(JPEG)不过编码器 → 可给满 5MP 2592×1944**(见 §5 实测)。

### P2 采用的编码尺寸(实测安全档)
| 宽高比设置 | 视频编码尺寸 | 说明 |
|---|---|---|
| 4:3(默认,满广角) | **1440×1080** | 探针已录出可播 HEVC;吃满传感器 4:3 广角 |
| 16:9 | **1920×1080** | 在 1920×1088 上限内 |

码率:20 Mbps 实测出片 ~6.9MB/3s(≈18Mbps 实际),清晰;P2 可按清晰度设置档位映射(如 1080P=20M)。

## 4. HEVC 端到端(Camera2 → MediaCodec → MediaMuxer)

- `recordClip(HEVC)` → **OK**,`/sdcard/FieldSight/_probe/probe_hevc.mp4` size=6,913,857,1440×1080,`video/hevc`,可播。
- 链路:`createEncoderByType("video/hevc")` + `COLOR_FormatSurface` + `createInputSurface()` → Camera2 `TEMPLATE_RECORD` `setRepeatingRequest([inputSurface])` → drain `dequeueOutputBuffer` → `MediaMuxer` `addTrack`/`writeSampleData` → `signalEndOfInputStream` → drain 尾。
- **降级路径**:`recordClip(useHevc=false)` AVC 同样可出片(HEVC 建/配置失败时 P2 走此路)。HEVC 主路无异常,降级未触发但代码就绪。

## 5. 三流并发 + 录像中拍照(核心解锁)

- `probeConcurrent()` → **streams=3 录像中拍照=成功 2592×1944 536170B**。
- 即:**编码器输入面 + JPEG ImageReader(满5MP)+ 预览 SurfaceTexture 三流共存**,`CaptureSession` 一次配成;录像中发一个 `TEMPLATE_STILL_CAPTURE`(target=jpegReader)成功落一张 **满 5MP 2592×1944** JPEG,录像不中断。
- **直接推翻 CameraX 旧限制**(只有 480P 双绑才能录像中拍照,720P/1080P 不行)。P2 全分辨率录像中拍照可行,拍照可给满 5MP。
- 代码含退化:三流配置抛异常则退 encoder+jpeg 双流;本机未触发。

## 6. GPS 元数据(MediaMuxer.setLocation)

- `muxer.setLocation(-36.85f, 174.76f)`(start 前调)→ mp4 `udta/©xyz` atom 实测写入 **ISO-6709 `-36.8499+174.7600/`**(float 精度致 .8500→.8499,可忽略)。
- **可写**。P2 用录制起点 GPS fix 写入;逐秒轨迹另存 `gps_track` 传后端 recordings 表(SP4 已备列)。

## 6b. 广角 / 变焦 / FOV 调查(回应"SMART-PTT 是不是 0.5x 超广角")

**结论:设备无 0.5x 超广角,SMART-PTT 只是把高度裁成 16:9。**

全部 cameraId 实测(`enumerateFov()`):

| cam | 朝向 | 物理尺寸 | 焦距 | 水平FOV | 对角FOV | activeArray | 变焦范围 |
|---|---|---|---|---|---|---|---|
| **0** | BACK(主) | 3.66×2.74mm | 3.75mm | **52.0°** | 62.7° | 2592×1944 | **[1.0, 4.0]** |
| 1 | FRONT | 2.9×2.18mm | 2.85mm | 53.9° | 65.0° | 2592×1944 | [1.0, 4.0] |
| 2 | BACK(副) | 2.9×2.18mm | 2.85mm | 53.9° | 65.0° | 2560×1440 | [1.0, 4.0] |

- **变焦下限全是 1.0** → 硬件根本不支持缩到 0.5x 超广角。无逻辑多摄(physicalCams 均空)。
- 后主 cam0 = **52° 水平 / 62.7° 对角**,普通广角(≈35mm 等效),**非超广角**。旧笔记"120° 广角"是未实测的错误假设,实测证伪。
- 后副 cam2 仅宽 ~2°(53.9°)且分辨率更低(2560×1440),不值得切换。

**直接量 SMART-PTT 原始录像对证**(设备 `/sdcard/DCIM/`):

| | 宽 | 高 | 画幅 | 分辨率 |
|---|---|---|---|---|
| SMART-PTT 照片 | 2560 | 1440 | 16:9 | 3.7MP |
| **本探针照片** | **2592** | **1944** | 4:3 | **5MP** |
| SMART-PTT 视频 | 1280 | 720 | 16:9 | 720P(hvc1) |
| **本探针视频** | **1440** | **1080** | 4:3 | **1080P**(hvc1) |

**宽度基本一致(同一颗 cam0,同吃满水平画幅,无横向裁剪);SMART-PTT 把高度裁成 16:9,画面反而更少。** 我们 4:3 默认 = 同宽 + 更高 + 更高分辨率,**严格优于 SMART-PTT**。P2 保持 4:3 默认即可,无需追"超广角"。

## 7. 对 P2 的直接结论

| 项 | 结论 |
|---|---|
| Camera2+MediaCodec HEVC 管线 | ✅ 可行,主路 HEVC |
| 视频最大分辨率 | 1080p 级(4:3=1440×1080,16:9=1920×1080),**不做满5MP 视频** |
| 满5MP 拍照 | ✅ 独立 JPEG ImageReader,不过编码器 |
| 全分辨率录像中拍照 | ✅ 三流并发,拍照可满5MP |
| 前台预览三流共存 | ✅(探针用 SurfaceTexture 假预览;P2 换真实 TextureView/SurfaceView) |
| HEVC 降级 H.264 | ✅ 代码就绪,AVC 同上限同可用 |
| setLocation GPS 元数据 | ✅ 写入 udta/©xyz |
| 宽高比设置 4:3/16:9 | ✅ 由编码尺寸切换(1440×1080 / 1920×1080)落地 |

**风险剩余(P2 逐项对照 SP3c 验收兜底)**:息屏后台三流会话维持、分段滚动时 muxer 切换不丢帧、预览挂/摘不断录、音频合流方案(复用现有 AudioRecorder 独立路径优先)、灯语/上传接线回归。这些是编排层回归,非相机能力硬伤。
